# 06. 여섯 번째 수업: 아무것도 없음을 나타내는 위대한 숫자 '0'

---

## 학습 목표
* 인류 역사상 가장 위대한 수학적 발견으로 꼽히는 '0'의 탄생 배경을 깨닫습니다.
* 0이 단순한 '없음(Nothing)'을 넘어, '자릿수를 메꾸는(Placeholder)' 엄청난 역할을 한다는 사실을 이해합니다.
* 파이썬의 `None`과 숫자 `0`이 컴퓨터 데이터에서 어떻게 완벽히 다르게 취급되는지 비교합니다.

## 1. 0은 처음부터 숫자가 아니었다

여러분에게 누군가 숫자를 세어보라고 하면 보통 "1, 2, 3, 4..." 하고 시작할 것입니다. "0, 1, 2..." 라고 세는 사람은 프로그래머밖에 없습니다!
인류에게 오랫동안 '0'이라는 숫자는 존재할 필요가 없었습니다. 사과가 한 개도 없다면, 그냥 "사과가 없어"라고 말하면 끝이니까요. 굳이 기호를 발명해서 "사과가 0개 있어"라고 말할 필요를 느끼지 못했습니다.

하지만 농업이 발달하고 교역이 일어나며 돈 계산이 복잡해지기 시작했습니다.
이집트나 바빌로니아 상인들은 점토판에 '205'라는 숫자를 적어야 할 때, 백의 자리에 2, 일의 자리에 5를 적었습니다. 그런데 십의 자리가 텅 비어있다는 것을 어떻게 표시했을까요?
초기에는 그냥 2와 5 사이를 넓게 띄워 썼습니다 (예: `2  5`). 하지만 띄어쓰기를 실수하면 그것은 25가 되기도 하고 2005가 되기도 했습니다. 

결국 기원전 바빌로니아 사람들은 **"여기는 빈칸입니다!"**를 의미하는 작은 쐐기 문자를 임의로 찍어 넣기 시작했습니다. 하지만 이 기호는 숫자라기보다는 그저 '띄어쓰기 쉼표' 같은 역할에 불과했습니다.

## 2. 인도에서 꽃피운 동그라미의 마법

"비어있음을 진짜 숫자로 다루자!"
이 파격적인 아이디어가 탄생한 곳은 바로 고대 **인도(India)** 였습니다. 
인도는 '공(空, 비어있음)'이라는 철학적 개념이 종교적으로 깊이 뿌리내린 곳이었습니다. 그들에게 '아무것도 없는 것'은 그냥 비어있는 것이 아니라 그 자체로 거대한 우주적 의미를 가진 상태였습니다.

인도 수학자들은 빈자리를 표시하기 위해 모래 위에 동그라미를 그렸고, 이것을 산스크리트어로 비어있다는 뜻의 **'슈냐(Shunya)'**라고 불렀습니다. 이 슈냐가 아라비아 상인들을 거쳐 유럽으로 넘어가면서 오늘날 우리가 아는 Zero(0)가 되었습니다.

## 3. 0의 두 가지 초능력

  <img src="./img/v3_07_lesson06_placeholder.png" alt="숫자들 사이의 간격을 지켜주는 히어로 0" width="500" />
</div>

인도에서 발명된 0은 현대 수학과 기수법을 폭발적으로 진화시킨 2가지 핵심 능력이 있습니다.

1. **상태 표시 (Nothingness)**: 양이 아무것도 없는 상태 그 자체를 나타내는 숫자로서의 가치. (예: $5 - 5 = \mathbf{0}$)
2. **자릿수 유지 (Placeholder)**: 10진법 위치기수법에서 빈자리에 들어가 다른 숫자들이 왼쪽으로 자리를 올릴 수 있도록 뼈대 역할을 해줍니다. (예: 1과 1 사이를 벌려 $\mathbf{101}$을 만듭니다.)
3. **연산 기호 (Operator)**: 어떤 숫자 오른쪽에 0을 하나 붙이면, 그 숫자는 순식간에 **10배**가 됩니다. (예: $23 \to 230$) 인도의 옛 시에는 "그녀 이마의 점은 그녀의 아름다움을 열 배로 만들어 준다"라는 구절이 있을 정도로, 0은 단순한 빈칸을 넘어 다른 숫자의 가치를 폭발시키는 연산자의 역할도 겸하게 되었습니다.

이 '0'의 발명 덕분에 이집트의 거대한 그림(연꽃)이나 로마 숫자(X, V)의 길고 불편한 나열이 완벽히 사라지고, 오늘날 전 세계가 0~9까지 단 10개의 기호만으로 모든 연산을 소화하는 **인도-아라비아 숫자(Hindu-Arabic Numeral System)** 체계를 완성하게 됩니다. (아라비아 상인들이 전파해서 아라비아 숫자라고 부를 뿐, 진짜 고향은 인도입니다!)

## 4. 파이썬의 빈자리: `0` vs `None`

컴퓨터 프로그래밍에서 0은 정말 특별합니다. 코딩 초보자들이 가장 많이 헷갈리는 것이 바로 수학적 숫자 `0`과 아예 데이터 자체가 비어버린 `None`의 차이입니다.

```python
# 1. 숫자 0 (인도의 위대한 '슈냐')
# 장바구니에 진짜로 '0원'짜리 무료 쿠폰이 담겨있는 상태!
price = 0
print(type(price)) # <class 'int'> (정상적인 정수 데이터 취급)

# 2. 파이썬의 None (진짜 아무것도 없음)
# 장바구니 자체가 삭제되었거나, 아예 가격표가 아직 붙지 않은 상태!
price_tag = None
print(type(price_tag)) # <class 'NoneType'>

# 만약 이 둘을 파이썬에게 더하라고 시키면 어떻게 될까요?
try:
    result = 0 + None
except TypeError as e:
    print("에러 발생! 0은 계산이 가능한 숫자지만, None은 숫자조차 아닙니다.")
```

0은 결코 무의미한 숫자가 아니라, 메모리 공간을 굳건히 차지하고 기수법 체계를 유지하는 '기둥'입니다. 이 0과 1이 만나면 드디어 현대 컴퓨터 기술의 궁극점인 2진법(Binary)이 깨어납니다. 다음 마지막 장에서 라이프니츠의 2진법과 컴퓨터 로직을 결합해 보겠습니다.

## 학습 정리
1. **0의 발명**: 고대 바빌로니아 상인들의 빈칸 표시를 넘어, 인도 수학자들이 철학적 '공(空)'의 개념을 바탕으로 '0(Zero)'을 진짜 하나의 숫자로 승격시켰습니다.
2. **인도-아라비아 기수법**: 0의 발명으로 빈자리에 대한 오류 문제가 해결되면서 위치기수법은 완벽해졌고, 단 10개의 숫자만으로 모든 계산을 지배하게 되었습니다.
3. **코딩의 `0`과 `None`**: 메모리 속에 당당히 자기 자리를 차지하고 연산이 가능한 데이터인 `0`과, 데이터 값이 아예 텅 비어 존재하는 상태인 `None`을 엄격히 구분해야 합니다.
