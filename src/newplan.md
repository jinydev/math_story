# 📚 미래형 수학 교재 (V3) 집필 기획서

우리는 전통적인 암기식 수학 교육을 탈피하여, 수학 본연의 논리와 현대 컴퓨터 과학(코딩, AI)을 융합한 차세대 수학 교재를 기획하고 있습니다.

---

## 1. 프로젝트 목적 및 방향성 (Core Vision)
* **목표**: 단순히 공식을 외우고 문제를 푸는 과거의 수학책이 아닌, 배운 원리가 현대 기술(인공지능, 데이터 과학 등)에서 어떻게 쓰이는지 직접 보여주는 **'살아있는 수학책'** 집필.
* **타겟 독자**: 
  * 정규 교과과목 수학을 예습/복습하려는 **초·중·고등학생**
  * 수학과 코딩의 연결 고리를 이해하고 싶은 **대학생 및 일반인**
* **차별화 포인트**: 기계적인 계산은 덜어내고, 원리의 탄생 배경과 컴퓨터(파이썬)가 이를 해결하는 방식을 나란히 비교하여 수학에 대한 흥미를 극대화합니다.

---

## 2. 콘텐츠 집필 가이드라인 (Content Strategy)

### 🚫 2.1. 저작권 완전 독립 및 OCR 오류 극복 (Complete Rewrite)
* 기존 시중 상업용 도서(예: "수학자가 들려주는~" 시리즈 등)의 텍스트 원본(`extracted/md`)은 오직 **목차와 뼈대 지식을 습득하는 용도**로만 활용합니다.
* **[중요] OCR 한계 인식과 딥 다이브**: 원본 OCR 추출물은 오탈자나 수식 누락이 매우 많고 완벽하지 않습니다! 이를 그대로 맹신하거나 단순 요약하지 말고, **에이전트 스스로의 강력한 수학적/컴퓨터공학적 지식을 적극적으로 주입하여 내용을 완전히 새롭게 창조하고 개선**해야 합니다.
* 특정 인물(수학자 등)이 화자가 되어 이야기하는 유치한 서술 방식이나 원래의 스토리텔링은 철저히 배제합니다. 

### 🏗️ 2.2. 구조화된 학습 흐름과 원본 목차 누락 방지 (Document Structure & TOC Integrity)
* **[중요] 원본 목차(ToC) 교차 검증**: 기존 추출된 원본 파일들의 텍스트만 대충 보고 임의로 챕터를 압축하거나 삭제하지 않습니다. 작업을 시작하기 전, 반드시 원본 도서의 실제 목차 이미지나 챕터 넘버링 체계를 꼼꼼히 확인하여 단 하나의 수업(챕터)도 누락되지 않도록 1:1 매칭 분할 구성을 최우선으로 원칙화합니다.
모든 개별 학습 마크다운(`.md`) 파일은 독자가 길을 잃지 않도록 반드시 다음 구조를 따릅니다.
1. **[도입부] 학습 목표 (Learning Objectives)**: 이 챕터에서 무엇을 배울 것인지 명확히 제시합니다.
2. **[본문] `##` 섹션별 시각화 필수**: 본문의 하위 섹션(`## 1.`, `## 2.` 등)에는 글만 빽빽하게 적지 않습니다. **가능하면 각 `##` 챕터마다 최소 1개 이상의 2D 생성형 이미지나 애니메이션 SVG를 반드시 배치**하여 시각적인 인지와 설명력을 극대화합니다.
3. **[결론] 학습 정리 (Summary)**: 챕터의 핵심 내용을 3~4줄로 명확하게 번호표를 매겨 요약합니다.

### 📐 2.3. 시각 자료 원칙 (Webtoon & SVGs)
* **2D 웹툰 스타일 이미지**: 중/고등학생들이 텍스트의 압박에서 벗어나도록 친근하면서도 시네마틱한 **2D 만화/웹툰 스타일**의 프롬프트를 작성하여 AI 이미지를 생성합니다.
* **마크다운 내 수식 표기**: 본문 내의 모든 수학 기호와 공식은 반드시 **LaTeX($$) 문법**을 사용하여 깔끔하게 렌더링되게 합니다.
* **[중요] SVG 렌더링 환경 주의사항**: 복잡한 수식이 연산되는 과정은 **웹 애니메이션 SVG**로 직접 하드코딩해 제작합니다. 단, **SVG 내부 영역은 HTML/브라우저가 LaTeX($$) 문법을 렌더링하지 못합니다.** 따라서 SVG 내부에 수식을 적을 때는 하드코딩된 일반 텍스트(`<text>`) 요소나 유니코드 기호를 사용해야 하며, 달러 기호($$)를 그대로 노출하여 텍스트가 깨지는 일이 없도록 각별히 주의합니다.
* **이미지 파일 분리 원칙**: 마크다운 파일 내부에 100줄이 넘어가는 우락부락한 SVG 코드를 직접 하드코딩하지 금지. 무조건(`./img/filename.svg`) 외부 파일로 분리 저장하여 연결(`![alt](./img/...)`)만 합니다.

### 💻 2.4. 파이썬(Python)과 수학의 완벽한 융합 (Coding Integration)
* **CS 개념의 브릿지**: 수학의 특정 단원이 **컴퓨터 과학(Computer Science)의 어떤 원리와 닿아있는지 논리적으로 연계**합니다. (예: 집합 = 파이썬 중복제거/데이터베이스 조인, 자연수/인덕션 = 재귀함수/클래스, 기수법 = 트랜지스터와 논리게이트 등)
* **파이썬 문법과 수학의 동질 확인**: 손으로 수식을 푸는 방식 바로 뒤에는, 똑같은 논리를 파이썬(Python) 코드로 어떻게 짜는지 **핵심 문법과 함께 설명**합니다. 변수, `while` 반복문, `if` 조건문, `set()`, `def` 함수 정의 등 수학과 1:1로 매칭되는 파이썬 쌩기초 문법 실습을 듬뿍 추가하여 코딩 교재의 역할까지 겸합니다.

## 3. 원고 구조 및 작업 프로세스 (Workflow)

집필 작업은 다음의 순서로 진행되며, 각 수학 교과 모듈(폴더) 단위로 체계적으로 관리합니다.

1. **원천 자료 추출**: `src/수학이야기 xx.주제명/extracted/` 폴더에서 OCR 뼈대 훑어보기. (오류 필터링 필수)
2. **신규 집필 폴더 준비**: `src/수학이야기 xx.주제명/` 모듈 루트에 `img` 하위 폴더 생성.
3. **마크다운(Markdown) 문서 집필**:
   * 영어 기반의 단정하고 직관적인 파일명 사용 (예: `00_intro.md`, `01_why_use_letters.md`).
   * 각 `##` 섹션마다 설명력 있는 텍스트와 풍부한 Python 코드 블록을 배치.
4. **시각 자료 별도 생성 (SVG/PNG)**: 
   * 각 섹션을 대변하는 2D 웹툰 이미지 생성.
   * 원리가 움직이는 SVG 폴리곤 렌더링 (LaTeX 절대 금지, 순수 태그만 사용)
5. **목차 연동**: `index.md`에 전체 학습 목표를 적고, 분리된 마크다운을 묶는 네비게이션 트리 작성.
6. **최종 검수**: 오타 수정 및 마크다운(LaTeX)/HTML 렌더링 확인, 파이썬 코드 블록의 동작 여부 검증.
