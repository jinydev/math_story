# 📚 미래형 수학 교재 (V3) 집필 기획서

우리는 전통적인 암기식 수학 교육을 탈피하여, 수학 본연의 논리와 현대 컴퓨터 과학(코딩, AI)을 융합한 차세대 수학 교재를 기획하고 있습니다.

---

## 1. 프로젝트 목적 및 방향성 (Core Vision)
* **목표**: 단순히 공식을 외우고 문제를 푸는 과거의 수학책이 아닌, 배운 원리가 현대 기술(인공지능, 데이터 과학 등)에서 어떻게 쓰이는지 직접 보여주는 **'살아있는 수학책'** 집필.
* **타겟 독자**: 
  * 정규 교과과목 수학을 예습/복습하려는 **초·중·고등학생**
  * 수학과 코딩의 연결 고리를 이해하고 싶은 **대학생 및 일반인**
* **차별화 포인트**: 기계적인 계산은 덜어내고, 원리의 탄생 배경과 컴퓨터(파이썬)가 이를 해결하는 방식을 나란히 비교하여 수학에 대한 흥미를 극대화합니다.

---

## 2. 콘텐츠 집필 가이드라인 (Content Strategy)

### 🚫 2.1. 저작권 완전 독립 (Complete Rewrite)
* 기존 시중 상업용 도서(예: "수학자가 들려주는~" 시리즈 등)의 이미지 트레이싱이나 스캔본에서 텍스트를 추출(`extracted/md` 활용)하여 뼈대 지식만 습득합니다.
* 특정 인물(수학자)이 화자가 되어 이야기하는 서술 방식이나 원래의 스토리텔링은 철저히 배제하고, **100% 새로운 예시와 현대적 관점(AI, IT)의 서술로 전면 재집필**합니다.

### � 2.2. 구조화된 학습 흐름 (Document Structure)
모든 개별 학습 마크다운(`.md`) 파일은 독자가 길을 잃지 않도록 반드시 다음 구조를 따릅니다.
1. **[도입부] 학습 목표 (Learning Objectives)**: 이 챕터에서 무엇을 배울 것인지 명확히 제시합니다.
2. **[본문] 원리 설명 및 파이썬 코드**: 직관적인 비유와 코딩 예제를 통해 개념을 설명합니다. 서술 시 가급적 수식을 말로 쉽게 풀어쓰되, 깊이 있는 이해가 필요한 본질은 타협하지 않습니다.
3. **[결론] 학습 정리 (Summary)**: 챕터의 핵심 내용을 3~4줄로 명확하게 요약하여 마무리합니다.

### 📐 2.3. 시각 자료와 2D 생성형 이미지 (Visual & Math Rendering)
* **2D 웹툰 스타일 이미지**: 생성형 AI 이미지는 기존의 무거운 3D가 아닌, 중/고등학생들이 좋아하는 친근하고 트렌디한 **2D 만화/웹툰 스타일**로 프롬프트를 작성하여 생성합니다. (예: `적분1/img/lesson6_negative_area.png` 참고)
* **수식 표기**: 모든 수학 기호와 공식은 반드시 **LaTeX($$) 문법**을 사용하여 깔끔하고 일관되게 렌더링되도록 작성합니다.
* **SVG 애니메이션 (계산 과정 시각화)**: 복잡한 수식이 어떻게 계산되고 결과가 만들어지는지 그 풀이 과정을 **애니메이션 SVG**로 제작하여 삽입합니다. 독자가 눈으로 수식의 메커니즘을 이해하도록 돕습니다.
* **이미지 파일 분리 원칙**: 마크다운 파일 내부에 길고 복잡한 SVG 코드를 직접 하드코딩하지 않습니다. 모든 SVG 그래픽과 생성형 AI 이미지는 각 모듈 폴더 내의 **`./img/` 디렉토리에 개별 파일로 분리 저장**한 뒤, 마크다운에서는 상대 경로(`![alt](./img/filename.svg)`)를 통해서만 깔끔하게 불러옵니다.

### 💻 2.4. 파이썬(Python) 코딩 융합 (Coding Integration)
* 손으로 공식을 푸는 방식을 설명한 직후, 파이썬 라이브러리(`SymPy`, `NumPy`, `Matplotlib` 등)를 이용해 컴퓨터로 똑같이 계산하거나 시각화하는 방법을 제시합니다.
* 코딩 예시는 수학이 실제 IT 산업에서 구동되는 방식임을 학생들에게 증명하는 핵심 무기입니다.

## 3. 원고 구조 및 작업 프로세스 (Workflow)

집필 작업은 다음의 순서로 진행되며, 각 수학 교과 모듈(폴더) 단위로 체계적으로 관리합니다.

1. **원천 자료 추출**: `src/수학이야기 xx.주제명/extracted/` 폴더에 기존 교재 텍스트 추출 보관.
2. **신규 집필 폴더 생성**: `src/수학이야기 xx.주제명/` (모듈 루트 폴더) 작업 착수 및 `img` 폴더 준비.
3. **마크다운(Markdown) 문서 집필**:
   * 영어 기반의 단정하고 직관적인 파일명 사용 (예: `00_intro.md`, `01_why_use_letters.md`).
   * 파일 상단에 '학습 목표', 하단에 '학습 정리' 필히 배치.
4. **시각 자료 별도 생성 (SVG/PNG)**: `./img/` 폴더에 생성형 이미지 및 계산 과정 SVG 애니메이션 파일 생성 후, 마크다운 본문에 링크 삽입.
5. **목차 연동**: `index.md`에 전체 학습 목표를 적고, 새롭게 정리된 영문 파일명의 경로를 이어주는 목차 링크 제공.
6. **최종 검수**: 오타 수정 및 마크다운(LaTeX)/HTML 렌더링 확인, 코드 블록 정상 작동 여부 꼼꼼히 체크.
